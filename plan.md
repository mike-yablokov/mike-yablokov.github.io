# План разработки сайта ymike.ru v2.0

## Общая информация

**Технологии:** Jekyll 4.3, Vanilla JS, CSS Grid/Flexbox  
**Рабочая папка:** `v2.0/`  
**Git ветка:** `v2.0` (новая ветка)  
**Хостинг:** GitHub Pages  
**Подход:** Desktop-first, красивые URL через Jekyll permalink  
**Языки:** Русская версия (английская - после полировки RU-версии)

**Контент:** Предоставляется в MD-файлах (главная, направления, кейсы)

---

## Этап 0: Подготовка репозитория

### 0.1 Создание новой ветки v2.0

```bash
git checkout -b v2.0
```

### 0.2 Проверка .gitignore

Убедиться что исключены из версионного контроля:
- `_site/`
- `.sass-cache/`
- `.jekyll-cache/`
- `.jekyll-metadata`
- `Gemfile.lock` (опционально)
- `extract_docx.py`
- `extracted_*.txt`
- `контент/` (папка с исходными docx)
- `.DS_Store`
- `Thumbs.db`

### 0.3 Начальный коммит

После проверки .gitignore сделать начальный коммит новой ветки.

## Этап 1: Настройка структуры Jekyll-проекта

### 1.1 Создание папок

```
v2.0/
├── _layouts/          # Шаблоны страниц
├── _includes/         # Переиспользуемые компоненты
├── _data/             # Структурированные данные
├── _cases/            # MD-файлы кейсов (коллекция Jekyll)
├── assets/
│   ├── css/          # Стили
│   ├── js/           # Скрипты
│   ├── images/       # Изображения (скопировать из photo/)
│   └── documents/    # PDF (скопировать из pdfs/)
├── index.md          # Главная страница
├── strategic.md      # Лендинг 1
├── social.md         # Лендинг 2
├── tourism.md        # Лендинг 3
├── digital.md        # Лендинг 4
└── education.md      # Лендинг 5
```

### 1.2 Настройка _config.yml

Обновить конфигурацию:

```yaml
title: "Михаил Яблоков"
description: "Эксперт по территориальной охране природы"
author: "Михаил Яблоков"

url: "https://ymike.ru"
baseurl: ""

markdown: kramdown
permalink: pretty  # Для красивых URL

plugins:
  - jekyll-sitemap
  - jekyll-seo-tag

# Коллекция для кейсов
collections:
  cases:
    output: false  # Кейсы не генерируют отдельные страницы
    permalink: /cases/:name/

# Exclude из билда
exclude:
  - extract_docx.py
  - extracted_*.txt
  - контент/
  - README.md
  - Gemfile
  - Gemfile.lock
  - vendor/
  - .sass-cache/
  - .jekyll-cache/

# Настройки Markdown
kramdown:
  input: GFM
  hard_wrap: false
  syntax_highlighter: rouge
```

### 1.3 Структура URL

Permalink для лендингов (в Front Matter каждого MD-файла):

- Главная: `/` (index.md)
- Направление 1: `/strategic/` (strategic.md)
- Направление 2: `/social/` (social.md)
- Направление 3: `/tourism/` (tourism.md)
- Направление 4: `/digital/` (digital.md)
- Направление 5: `/education/` (education.md)

Для английской версии (в будущем):
- `/en/`
- `/en/strategic/`
- и т.д.

### Порядок подготовки изображений (вручную)

**Этап 8 (Оптимизация изображений)** - пользователь подготовит самостоятельно:

#### Шаг 1: Ручная обрезка и композиция

**Важно:** Фотографии требуют ручной обрезки для правильной композиции!

Для каждого фото создать 3 версии с учетом композиции:
- **Desktop** - широкий кадр, горизонтальная композиция
- **Tablet** - промежуточный размер, та же композиция
- **Mobile** - может потребоваться перекадрирование (вертикально для hero)

**Инструменты для обрезки:**
- Adobe Photoshop / Lightroom
- GIMP (бесплатно)
- Affinity Photo
- Photopea.com (онлайн, бесплатно)

#### Шаг 2: Экспорт в нужных размерах

После обрезки экспортировать в размерах из таблицы выше.

**Формат экспорта:**
- Основной: **WebP** (качество 85%)
- Fallback: **JPEG** (качество 85%)

**Именование файлов:**
```
ymike.ru_0.1-1920w.webp    (desktop)
ymike.ru_0.1-1280w.webp    (tablet)
ymike.ru_0.1-768w.webp     (mobile)
ymike.ru_0.1-1920w.jpg     (fallback для старых браузеров)
```

#### Шаг 3: Конвертация в WebP

**Если редактор не поддерживает WebP напрямую:**

**Онлайн-конвертеры (проще всего):**
- [Squoosh.app](https://squoosh.app/) - Google, WebP конвертер (рекомендуется)
- [CloudConvert](https://cloudconvert.com/jpg-to-webp) - пакетная конвертация
- [Convertio](https://convertio.co/ru/jpg-webp/) - до 100MB бесплатно

**Настройки при конвертации:**
- Качество: 85%
- Формат: WebP
- Цветовое пространство: sRGB
- Метаданные: можно удалить (для уменьшения размера)

#### Шаг 4: Организация файлов

Разложить подготовленные фото по папкам:

#### Шаг 5: Проверка размеров файлов

**Целевые размеры (WebP):**
- Hero desktop: 150-250KB
- Hero tablet: 80-120KB
- Hero mobile: 40-60KB

### Количество файлов для подготовки

**Всего изображений для обработки:**

1. **Hero-секции:** 6 фото × 3 размера × 2 формата = 36 файлов
2. **Эмоц. перебивка:** 1 фото × 3 размера × 2 формата = 6 файлов
3. **Карточки направлений:** 5 фото × 1 размер × 2 формата = 10 файлов
4. **Фото-секции лендингов:** 10 фото × 3 размера × 2 формата = 60 файлов
5. **Карточки кейсов:** ~15 фото × 3 размера × 2 формата = 90 файлов
6. **Логотипы:** ~10 логотипов × 1 размер = 10 файлов (SVG или PNG)

### 6.3 JavaScript для модальных окон

**Файл:** `assets/js/modal.js`

**Функционал:**
- Открытие по клику на карточку кейса (data-modal-open="case_1_0")
- Закрытие:
  - Клик на крестик
  - Клик на оверлей
  - Нажатие Escape
- Блокировка скролла body при открытом модале
- Trap focus внутри модального окна (доступность)

```javascript
document.addEventListener('DOMContentLoaded', function() {
  const modalTriggers = document.querySelectorAll('[data-modal-open]');
  const modalCloseButtons = document.querySelectorAll('[data-modal-close]');
  
  modalTriggers.forEach(trigger => {
    trigger.addEventListener('click', function(e) {
      e.preventDefault();
      const modalId = this.getAttribute('data-modal-open');
      const modal = document.getElementById(`modal-${modalId}`);
      openModal(modal);
    });
  });
  
  modalCloseButtons.forEach(btn => {
    btn.addEventListener('click', function() {
      const modal = this.closest('.modal');
      closeModal(modal);
    });
  });
  
  document.addEventListener('keydown', function(e) {
    if (e.key === 'Escape') {
      const openModal = document.querySelector('.modal[aria-hidden="false"]');
      if (openModal) closeModal(openModal);
    }
  });
  
  function openModal(modal) {
    modal.setAttribute('aria-hidden', 'false');
    document.body.style.overflow = 'hidden';
  }
  
  function closeModal(modal) {
    modal.setAttribute('aria-hidden', 'true');
    document.body.style.overflow = '';
  }
});
```
