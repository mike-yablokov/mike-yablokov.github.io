# ymike.ru v2.0

Официальный сайт Михаила Яблокова — эксперта по территориальной охране природы.

## О проекте

Сайт собран на Jekyll и размещается на GitHub Pages с использованием собственного домена `ymike.ru`.

### Основные возможности

- Главная страница с презентацией экспертизы и направлений работы
- 5 лендингов по направлениям: стратегии, социальные решения, туризм, цифровые решения, обучение
- Кейсы в модальных окнах (данные в YAML, рендер через JavaScript)
- Полная адаптивность (desktop-first верстка с CSS Grid/Flexbox)
- Две языковые версии (RU/EN) с отдельными страницами и общим layout
- Интеграция Яндекс.Метрики и формы консультации на базе Яндекс.Форм

### Технологии

- **Jekyll 4.3** — статический генератор сайта
- **Vanilla JavaScript** — интерактивность, модальные окна, поведение переключателя языка
- **CSS Grid/Flexbox** — адаптивная верстка
- **GitHub Pages** — хостинг
- **Yandex Metrica / Yandex Forms** — аналитика и форма записи на консультацию

## Локальная разработка

### Требования

- Ruby 2.7+
- Bundler
- Jekyll 4.3+

### Установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/YOUR_GITHUB_USERNAME/ymike.ru.git
cd ymike.ru
```

2. Установите зависимости:
```bash
bundle install
```

3. Запустите локальный сервер:
```bash
bundle exec jekyll serve
```

4. Откройте в браузере: http://127.0.0.1:4000

### Команды разработки

```bash
# Запуск сервера
bundle exec jekyll serve

# Сборка сайта
bundle exec jekyll build

# Сборка с черновиками
bundle exec jekyll serve --drafts

# Очистка кэша
bundle exec jekyll clean
```

## Деплой на GitHub Pages

Сайт деплоится на GitHub Pages из ветки `main`.

- Продакшн-URL: `https://ymike.ru`
- В `_config.yml` указано:
  - `url: "https://ymike.ru"`
  - `baseurl: ""` (сайт в корне домена)
- Для кастомного домена используется стандартный механизм GitHub Pages (`CNAME` с `ymike.ru` и DNS A-записи на IP GitHub).

## Контент и мультиязычность

- Все основные страницы существуют в двух вариантах: русский (`/*.md`) и английский (`/en/*.md`).
- Переключатель языков в `header.html` переводит пользователя на эквивалентную страницу (`/page/` ↔ `/en/page/`) и старается сохранить позицию прокрутки.
- Тексты кейсов хранятся в `_data/modal-cases.yml` (RU) и `_data/modal-cases-en.yml` (EN) и рендерятся в модальных окнах через `assets/js/modal.js`.

## Аналитика и формы

- Яндекс.Метрика подключена через стандартный скрипт (`_includes/head.html`) с `site.metrika_id` из `_config.yml`.
- Кнопки «Записаться на консультацию» открывают модальное окно с Yandex.Forms (`_includes/consult-modal.html`), логика открытия/закрытия реализована в `assets/js/main.js`.
